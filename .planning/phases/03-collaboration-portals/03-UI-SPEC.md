## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, avatar-to-text gaps |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, page content `p-6` (established pattern) |
| xl | 32px | Layout gaps between major sections, comment thread indentation (`pl-8`) |
| 2xl | 48px | Page-level vertical separation |
| 3xl | 64px | Not used in this phase |

No exceptions. Comment thread indentation for nested replies uses 32px (`pl-8`) from the standard scale.

Source: Existing codebase uses `space-y-6`, `gap-4`, `p-6`, `pb-2`, `gap-1.5` consistently.

|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / oklch(0.985 0 0) | Page background, surfaces, portal backgrounds |
| Secondary (30%) | `--card` / oklch(1 0 0) | Cards, comment boxes, timeline containers, sidebar nav |
| Accent (10%) | `--primary` / oklch(0.457 0.24 277.023) / indigo-600 | See reserved list below |
| Destructive | `--destructive` / oklch(0.577 0.245 27.325) | Delete note, remove comment only |

Accent reserved for:
- "Plaats reactie" / "Notitie toevoegen" primary CTA button
- @mention highlight chips in comment text
- Active portal nav item indicator
- Unread notification dot
- "Goedkeuren" action in client portal

Accent is NOT used for: timestamps, read-only badges, metadata labels, separator lines.

## Interaction Contracts

### Comment Thread

| Interaction | Behavior |
|-------------|----------|
| Type `@` in comment input | Opens `Command` popover anchored below cursor position. Shows team members filtered by typed characters. |
| Select @mention | Inserts `MentionChip` (indigo badge) into textarea at cursor position. Popover closes. |
| Click "Plaats reactie" | POST comment to API. Optimistic insert into thread. Loading spinner on button during request. |
| Toggle "Intern" | Switches comment visibility. Internal = only team members see it. Shared = visible in portal. Visual indicator: `Badge variant="outline"` with lock icon for internal. |
| Edit comment (own only) | Inline edit mode: textarea replaces rendered text. "Reactie opslaan" / "Bewerking annuleren" buttons appear. |
| Delete comment (own only) | Confirmation dialog: "Weet je zeker dat je deze reactie wilt verwijderen?" with destructive button. |

### Activity Timeline

| Interaction | Behavior |
|-------------|----------|
| New activity arrives (Socket.IO) | Prepend to timeline with subtle fade-in animation (150ms ease-in). |
| Timeline exceeds 20 items | Show "Toon oudere activiteiten" collapsible. Load more on expand. |
| Click actor name in timeline | Navigate to relevant entity (candidate/vacancy). |

### Portal Approval Flow (Client Portal)

| Interaction | Behavior |
|-------------|----------|
| Click "Goedkeuren" | PATCH candidate status to `approved`. Optimistic update: badge turns green. |
| Click "Afwijzen" | Opens dialog: "Reden voor afwijzing?" with textarea. Required field. On submit, PATCH status to `rejected`. |
| View candidate detail | Opens Sheet/drawer from right. Read-only candidate info + activity timeline showing shared (non-internal) comments only. |

### Real-Time Updates

| Event | UI Response |
|-------|-------------|
| New comment on viewed entity | Append to thread. If comment from other user, show subtle blue dot indicator. |
| Candidate status change | Update badge color in portal list. If portal user viewing that candidate, update inline. |
| New activity on viewed vacancy | Prepend to timeline with animation. |

## State Management

| State Type | Tool | Scope |
|------------|------|-------|
| Comment thread data | TanStack Query | Per entity (candidate/vacancy ID) |
| Activity timeline data | TanStack Query | Per entity, infinite query for pagination |
| Portal vacancy list | TanStack Query | Scoped to authenticated portal user |
| Real-time comment updates | Socket.IO + Zustand | Connection status in Zustand, invalidate TanStack Query cache on event |
| Comment input draft | React Hook Form (local) | Per comment input instance |
| @mention search | TanStack Query | Debounced user search |
| Portal filter/sort state | nuqs | URL params for portal list views |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>= 1024px) | Sidebar visible. Collaboration panel as right column or tab panel alongside main content. Portal uses full card grid. |
| Tablet (768-1023px) | Sidebar collapsible (existing behavior). Collaboration panel moves to full-width tab below main content. Portal cards stack to 2 columns. |
| Mobile (< 768px) | Sidebar collapsed to icon rail. Collaboration as full-page tab. Portal cards single column. Comment input sticky at bottom. Candidate detail opens as full-screen sheet. |
